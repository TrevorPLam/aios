# Product Documentation

## Plain English Summary

Product documentation bridges engineering and product management. It includes PRDs (Product Requirement Documents), user journeys, and acceptance criteria that define what we're building and why.

## Documentation Types

| Type | Purpose | Audience |
| ------ | --------- | ---------- |
| **PRD** | Define what to build | Eng, PM, Design |
| **User Journeys** | Show how users interact | Eng, Design, PM |
| **Acceptance Criteria** | Define "done" | Eng, QA, PM |

## Process

1. **PRD Created** - PM writes PRD using template
2. **Technical Review** - Engineering reviews feasibility
3. **Design Review** - Design creates mocks
4. **Implementation** - Engineering builds feature
5. **Acceptance** - QA verifies against criteria
