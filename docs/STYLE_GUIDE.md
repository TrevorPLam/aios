# AIOS documentation style guide

## Voice and tone
- Use active voice: "The system processes requests".
- Use present tense: "The button opens".
- Write in second person: "You can configure".

## Formatting
- Use sentence-case headings.
- Prefer short, scannable paragraphs.
- Use ordered lists for sequences and unordered lists for options.
- Use fenced code blocks with language identifiers.

## Terminology
- AIOS (all caps)
- AsyncStorage (camelCase)
- Command Center (title case)
- React Native (title case)

## File conventions
- Root-level docs: UPPERCASE.md
- Technical docs: kebab-case.md
- Use descriptive names (avoid "misc" or "notes").

## Links
- Use relative links for internal references.
- Use absolute links for external references.
- Include the .md extension for internal markdown links.

## Examples

### Preferred
"You can restart the server after updating environment variables."

### Avoid
"The server will be restarted after environment variables are updated."
