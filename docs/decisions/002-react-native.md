# ADR-002: Use React Native for mobile client

**Status:** Accepted  
**Date:** 2026-01-16

## Context
AIOS targets mobile-first workflows with shared logic across iOS and Android. The team needs a strong ecosystem, familiar tooling, and a reliable community.

## Decision
Use React Native as the primary mobile framework.

## Consequences
- Enables a single codebase across iOS and Android.
- Leverages TypeScript, React patterns, and existing team expertise.
- Requires native dependency management and platform-specific testing.
- Keeps alignment with Expo tooling and ecosystem packages.
