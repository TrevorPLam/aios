# AGENT.md (Folder-Level Guide)

## Purpose of this folder
Root repository folder - contains configuration files, top-level documentation, and primary project structure.

## What agents may do here
- Create or update configuration files (package.json, tsconfig.json, etc.)
- Update root-level documentation (README.md, SECURITY.md, etc.)
- Add or modify governance and policy files in .repo/
- Update priority TODO files (P0TODO.md, P1TODO.md, etc.)

## What agents may NOT do
- Delete critical files without explicit justification
- Bypass governance structure defined in .repo/policy/
- Modify files that require HITL approval without proper workflow

## Required links
- Refer to higher-level policy: /.repo/policy/CONSTITUTION.md
- Governance structure: /.repo/policy/
- Standards: /.repo/docs/standards/
